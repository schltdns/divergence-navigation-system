Beitragsrichtlinien
Vielen Dank für dein Interesse, zu diesem Projekt beizutragen.

Ich entwickle dieses Framework als Einzelperson. Beiträge sind willkommen, solange sie die Integrität, Klarheit und epistemische Ausrichtung des Projekts respektieren.

Wie du beitragen kannst
Ich akzeptiere Beiträge in folgenden Formen:

Fehlermeldungen (Issues)  
Wenn du Fehler, Unklarheiten oder Inkonsistenzen findest, eröffne bitte ein Issue mit einer klaren Beschreibung.

Verbesserungsvorschläge  
Vorschläge zu Struktur, Dokumentation oder methodischer Klarheit sind willkommen.

Pull Requests  
Wenn du Änderungen vorschlägst, erstelle bitte einen Pull Request mit einer kurzen Begründung und klaren Commit‑Nachrichten.

Grundsätze für Beiträge
Bitte beachte beim Mitwirken:

Respektvoller und konstruktiver Umgang

Präzise, nachvollziehbare Formulierungen

Keine spekulativen Erweiterungen ohne methodische Grundlage

Keine Änderungen, die die epistemische Struktur (P1–P8) verzerren

Keine kommerziellen Erweiterungen (Lizenz: Non‑Commercial)

Ablauf für Pull Requests
Forke das Repository

Erstelle einen neuen Branch

Führe deine Änderungen durch

Beschreibe klar, was du geändert hast und warum

Reiche den Pull Request ein

Ich prüfe jeden Beitrag sorgfältig und gebe Rückmeldung, sobald es mir möglich ist.

Kontakt
Für Fragen oder Hinweise kannst du ein Issue eröffnen.
